一、VI与VIM
1、vim的三种模式
普通模式:用户刚刚启动Vi/Vim则进入普通模式，此状态下敲击键盘动作会被Vim识别为命令，而非输入字符插入模式:普通模式按输入i之后，进入插入模式命令行模式:普通模式按冒号进入命令行模式	
2、模式切换：
   普通模式进入插入模式：i a o I A O
   插入模式进入普通模式：esc
   普通模式进入命令行模式：输入 :
   命令行模式进入普通模式：删除 :
   命令行模式与插入模式不能直接交互。
3、vim命令(进入普通模式)
    vim filename: 打开或新建文件filename，将光标置于第一行行首
    vim +n filename: 打开文件filename，并将光标置于第n行行首
    （n:为数字）    vim + filename: 打开文件filename，并将光标置于最后一行行首
4、普通模式进入插入模式:
  i：在光标所在位置前插入文本
  I：将光标移动当前行的行首，然后再其前插入文本  a：用于在光标当前所在位置之后追加新文本	
  A：将光标移到所在行的行尾，从那里开始插入新文本  o：在光标所在行的下面新创建一行，并将光标置于行首，等待输入文本  O(大写字母o)：在光标所在行的上面创建一行，并将光标置于行首，等待输入文本
5、普通模式下光标命令
   G ：将光标移至最后一行行首   nG ：将光标移至第n行行首 （n:为数字）   n+ ：将光标下移n行  （n:为数字）   n- ：将光标上移n行  （n:为数字）   n$ ：光标下移n-1行行尾  （n:为数字）   0（数字0）：将光标移到所在行的行首   $ ：将光标移到所在行的行尾   ^ ：将光标移动所在行的第一个字符（非空字符）
   h,j,k,l ：分别用于光标左移，下移，上移，右移一个字符   H ：将光标移至当前屏幕首行的行首   M ：将光标移至屏幕显示中间行的行首   L ：将光标移至当前屏幕最低行的行首

6、普通模式下操作的命令
   x ：删除光标处的字符
   dw ：删除一个单词
   dd ：删除光标所在行
   yy ：复制光标所在行
   p ：粘贴
   /str ：向右进行搜索 （str:为字符串）
   ?str ：向左进行搜索 （str:为字符串）
   n（英文字母n） ：向相同的方向查找
   N ：向不同的方向查找

7、命令行模式
   :q    :  退出
   :wq   :  保存并退出
   :q!   :  强制退出不保存

8、当vi或vim因为系统异常，导致崩溃，强制退出，之前正在编辑的内容会被保存下来，但保存的文件会被放在交换区swp中。假设之前文件名称为a.md,保存下来的文件名为a.md.swp。如果我们想要打开之前的a.md,那么需要打开文件a.md.swp，进入到a.md.swp在文件下方会有S(ave)、D(elete)...,我们可以根据提示进行相应操作，例如S(保存)或D(删除)，执行完操作之后，系统会将a.md进行保存或删除，当操作结束后，记得将.swp的文件删除，否则下次进入依旧会进入到.swp的文件。

9、常用命令-文件相关命令
  9.1、cat：显示文件内容，正序展示
     cat filename
     cat -n filename   （n为字母n） 显示行号
     cat -b filename  显示行号(将空行的行号删掉)
  9.2、tac：显示文件内容，倒序展示
     tac filename
  9.3、head：展示文件部分内容，默认展示前十行
     head filename  展示文件部分内容，默认展示前十行
     head -n filename 展示文件部分内容，展示前n行  n为数字
  9.4、tail：展示文件部分内容，默认展示后十行
     tail filename  展示文件部分内容，默认展示后十行
     tail -n filename 展示文件部分内容，展示后n行  n为数字
  9.5、more：展示文件部分内容，空格翻页
     当文件内容很少，在终端可以完全展示，more好似无效。
  9.6、wc： 计算文件内容
     行数  单词数  字符数  文件名称


二、安装Java环境
1、需要一个java的源码安装包  .tar.gz
2、解压到我们常用的而且短时间不会删除的目录中
   tar -zxvf java压缩包名称 -C 指定目录
3、进入到jdk的解压文件的bin下测试java是否可用  ./java
4、将java配置到环境变量中：
    进入环境变量的配置文件中： vim + ~/.bashrc  
    注：不要删除源文件的任何内容
5、导出环境变量的路径  export (导出或创建一个变量)
   export JAVA_HOME=/home/用户名/software(我们自己创建的一个目录)/jdk-10.0.1    创建了一个名为JAVA_HOME的变量，该变量为jdk的解压缩路径
   export CLASSPATH=.  java命令执行时查找class文件的路径为当前目录
    export PATH=$JAVA_HOME/bin:$PATH   设置新的环境变量

6、$JAVA_HOME/bin:$PATH
  $变量名：引用之前已经创建好的变量，类似windows下的 %路径名%
  : ：类似windows下的; ， 分隔不同的文件路径
  $PATH ：将之前的已经存在的环境变量保留并拼接 

7、刷新配置文件
    source ~/.bashrc


三、Linux下环境变量的作用域
1、临时变量：只在当前终端命令窗口有效
2、用户变量：在当前用户下的任意目录下均有效   /home/用户名/.bashrc  
3、系统变量：在当前系统下的任意目录下均有效  /etc/profile

六、创建用户
1、图形化界面创建一个新用户：
all setting -> user account -> 右上角解锁 -> 左下角+ ->创建新的用户

2、命令创建用户
注：在Linux系统下，每个用户必须在某一个组下，没有组的用户没有任何意义，属于无效的用户。创建用户时没有指定组，系统会默认分配一个组，该组的名称与用户名一致。
用户：可以进入系统的一对账号及密码
组  ：每个用户至少拥有一个组，可以使用组设置用户权限，一个组拥有的权限，该组下的所有用户均有该权限。

用户相关操作
2.1、创建新用户
   sudo useradd -m 用户名
   -m ：会自动在home目录下创建一个与用户名一致的用户目录(该目录为用户登入目录) 
2.2、创建的新用户需要设置密码
   sudo passwd 用户名
2.3、查看用户所属群组
   groups 用户名
  例子： groups lily
         ==> lily : lily  ==> 用户名:主组
         groups lily
         ==> lily : lily LOL  ==> 用户名:主组 附加组
2.4、修改用户
   sudo usermod -G 组名 用户名
例子： sudo usermod -G LOL lily
      （当前用户会有两个组， LOL为附加组）
   sudo usermod -g 组名 用户名
例子： sudo usermod -g WOW lucy
      （当前用户的主组会更改，将WOW作为lucy主组）
注：使用群组-G或-g时，尽量使用-G,-G会将修改的组作为该用户的附加组使用，附加组为主组的附属品，可以随时被删除；如果使用-g，会修改主组，主组在删除时，必须保证该下没有任何用户才能删除，如果主组中有用户，会删除失败。
2.5、删除用户
  sudo userdel 用户名  （创建用户时自动生成的文件目录不会删除，只是将用户删除了） 
  sudo userdel -r 用户名 （创建用户时自动生成的文件目录会一起清除干净，以后尽量使用-r） 


组的操作
2.6、查看当前系统下存在的组：
   文件为: /etc/group
2.7、创建组
    sudo groupadd 组名
2.8、删除组
    sudo groupdel 组名
    (注：当组为主组时。如果主组下有用户，删除失败)

3、用户与组
a、组
   主组：用户必须存在一个主组，改组在删除时，用户必须清空
   附加组：可以在任意状态下删除，删除附加组后，用户与该组的关系自动解除，权限自动消失。
b、用户：
   默认创建的用户在home下没有登入目录，创建时需要使用-m选项创建登入目录   sudo useradd -m 用户名

c、组与用户的关系：
  一个组可以有多个用户
  一个用户可以有多个附加组

七、文件系统
1、在不同操作下，使用的文件格式是不同的
   windows : ntfs
   mac   : hfs
   linux : ext2  ext3  ext4
fat32:在各个平台下均能使用，小于4G；大于4G，使用fat64

2、Linux中的文件特性：
   Linux下所有的文件都是通过权限设置的。
2.1、权限分为十位
  第一位为类型
  - : 文件
  d : 目录
  l : 链接

  后九位，每三位为一组
  第一组为当前用户的权限
  第二组为当前组的权限
  第三组为其他组的权限
 - : 0
 x : 1
 w : 2
 r : 4
通过四个符号可以组成0-7  8个数

3、修改文件权限
3.1、修改文件权限命令
   chmod [ugoa][+-][rwx] 文件或文件夹
用户选项说明：
  u user用户  g group 组
  o other 其他用户  a all 所有用户权限操作选项
  + 增加权限  - 删除权限

权限说明
  r 读权限  w 写权限  x 执行权限
3.2、修改文件权限命令
   chmod n1n2n3 文件或文件夹
数值设置：
   n1:当前用户权限
   n2:当前组的权限
   n3:其他组的权限
设置的数值取值范围为 0-7
例子： 所有的用户都只保留可执行权限
      chmod 111 print.md
      当前用户有所有权限，当前组由可读可执行权限，其他无权限
      chmod 750 print.md
